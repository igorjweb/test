O que e ?

Um teste unitário é uma técnica de desenvolvimento de software em
que pequenas partes do código (geralmente chamadas de unidades)
são testadas isoladamente para garantir que funcionam como esperado.

Caracteristicas

  • Isolamento: Testa apenas uma funcionalidade ou componente
    específico.

  • Automação: Deve ser possível rodar o teste
    automaticamente, sem intervenção manual.

  • Previsibilidade: Um teste unitário bem escrito
    sempre retorna o mesmo resultado para os
    mesmos dados de entrada.

Etapas

  • Preparação: Configura o ambiente para o teste,
    criando dados ou simulando componentes.

  • Execução: Chama a unidade de código sendo testada,
    com entradas específicas.

  • Validação: Compara o resultado obtido com o esperado.

Exemplos

Explicação: Testes unitários verificam pequenas partes do código
(funções ou métodos) para garantir que funcionem corretamente.
Eles são automáticos rápidos e ajudam a identificar
erros de forma antecipada.
